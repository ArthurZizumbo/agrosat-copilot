### Decisiones tecnicas no obvias (Fase A)

- **`NotebookCard` dataclass frozen** en `ml/report/notebook_content.py` es la
  fuente unica de verdad: dashboard Streamlit (`render_card`) y plantilla
  Jinja2 (`{% for card in cards %}`) consumen el mismo objeto.
- **Conclusiones citan numeros reales** del EDA (OOB 0.888 Italia, 0.831 Francia,
  CV NDVI 53.9%, 545 timesteps inválidos sobre 8,600, etc.) — no son texto
  genérico. Cualquier cambio al contenido del EDA requiere actualizar la
  ficha correspondiente.
- **`_slugify` + `_nearest_heading`** en el extractor genera nombres
  semánticos a partir del heading markdown más cercano. Si el notebook
  reordena celdas, los nombres cambian — es esperado: el orden alfabético
  del PDF refleja el orden semántico del notebook.
- **Aliases legacy `render_tab_sentinel2/alphaearth/bivariate/pastis/global`**
  preservados con firma `(figures_dir: Path | None = None)` para no romper
  tests externos ni el AppTest histórico.
- **Ficha "globales" (`GLOBAL_CARD.figures_dir = ""`)**: no tiene figuras
  asociadas; `list_figures` retorna lista vacía y el template muestra sólo
  el bloque de conclusiones. No genera placeholder de figuras pendientes.
- **Compatibilidad de tests**: `tests/app/test_dashboard.py` ahora usa
  `pytest.importorskip("streamlit.testing.v1").AppTest` y skipea la suite
  completa si streamlit no está instalado — mantiene contrato del handoff
  original.
- **Reproducibilidad PDF**: orden alfabético de PNGs por carpeta sigue garantizando
  PDF bit-identical entre runs (mantenido desde versión anterior).
