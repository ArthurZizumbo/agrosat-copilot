## 0. Resumen ejecutivo

US-011 entrega el **notebook secuencial `notebooks/02b_eda_alphaearth.ipynb`** que caracteriza las **64 dimensiones del embedding AlphaEarth Foundations v2.1** (Google DeepMind, GEE colección `GOOGLE/SATELLITE_EMBEDDING/V1/ANNUAL`) sobre las 3 ROIs italianas piloto (Pianura Padana, Toscana centrale, Apulia). El notebook es la justificación empírica de por qué AlphaEarth es el backbone principal de features (decisión irrevocable §AGENTS.md) y de qué subset de dimensiones merece priorización para el baseline US-017.

**Decisión pragmática (12-may-2026, alineada con US-010)**: los embeddings se obtienen **on-the-fly desde GEE vía `sampleRegions` server-side**, no se persisten en disco en esta US. La descarga productiva a GCS + asset Dagster + tabla PostGIS `alphaearth_tiles` se cierra en US-006 (sprint S3). Para el EDA basta con muestreo estratificado de **100k píxeles por región y clase** según pide el plan v6 línea 1020.

**Labels para coloreado y feature importance — Patrón A (validación cruzada Italia ↔ Francia)**: el plan v6 pide GSAA italiano sobre las 3 ROIs italianas. GSAA Italia no está disponible (depende de US-008). Resolución acordada 12-may-2026:

1. **Sección 1 del notebook (cuerpo principal · cubre rúbrica literal)**: AlphaEarth sobre las **3 ROIs italianas** + **Dynamic World v1** (`GOOGLE/DYNAMICWORLD/V1`, 9 clases LULC globales) como labels proxy. Disponible inmediato vía GEE, sin descarga.
2. **Sección 2 del notebook (validación cruzada · valor agregado)**: AlphaEarth sobre los **2,433 patches PASTIS-R en Francia** (ya descargados en `data/PASTIS-R/`) + **labels reales de cultivo PASTIS-R** (**20 clases canónicas**: 0 background + 1-18 cultivos + 19 void, en canal 0 de `ANNOTATIONS/TARGET_*.npy` shape `(3, 128, 128)`). Las coordenadas se extraen reproyectando `metadata.geojson` de EPSG:2154 → EPSG:4326. **Rango temporal AlphaEarth**: dado que PASTIS-R cubre 2018-09 a 2019-11, samplear AlphaEarth **año 2019** (anual agregado que cubre la ventana real de PASTIS) para evitar confound temporal. Lookup table en [`data/reference/pastis_class_mapping.json`](../../data/reference/pastis_class_mapping.json) (creado 12-may con 3 agrupaciones derivadas + metadata oficial).
3. **Análisis paralelo, NO concatenado**: las dos secciones corren los mismos análisis (correlación, distribución, t-SNE, RF importance) **por separado**. Se evita explícitamente mezclar samples Italia + Francia en un único modelo / scatter para no confundir señal de clase con señal de dominio geográfico (clima mediterráneo vs continental).
4. **Conclusión final**: tabla comparativa de las **top-10 dims AlphaEarth en Italia vs Francia** con columna `consistente_en_ambas_regiones (sí/no)`. Esa tabla alimenta directamente US-017 (decidir si el baseline necesita regionalización).
5. **GSAA Italia** queda deuda explícita a cerrar en US-008 antes del Avance 3 (20-may).

**Razón del Patrón A** (decisión expert 12-may, recogida en §10): mezclar Francia + Italia en un único t-SNE/RF introduce **domain shift confound** que invalida conclusiones sobre separabilidad de clases. Patrón A da al evaluador (a) cobertura literal de la rúbrica sobre Italia, (b) validación independiente sobre dataset benchmark con labels cultivar-específicos, (c) evidencia explícita de generalización del embedding. Costo: +1 SP, ~1.5 h.

US-011 reutiliza al máximo los helpers de US-010 (`ml/utils/sampling.py`, `ml/analysis/visualization.py`, `ml/ingest/gee_sampler.py`) + extiende `ml/ingest/pastis_loader.py` con extracción de coordenadas geo desde `metadata.geojson` + añade módulo nuevo `ml/analysis/embeddings.py` para reducción dimensional, correlación, feature importance, estabilidad temporal y **cross-region comparison**.

def pastis_pixel_labels(
    patch_id: str,
    root: Path | None = None,
    sample_per_patch: int | None = None,
    seed: int = 42,
    exclude_classes: tuple[int, ...] = (0, 19),
