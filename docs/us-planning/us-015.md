### 2.1 Decisión técnica clave — separar `parcels` y `features_parcels`

El `db/CLAUDE.md` describe `parcels (id, session_id, aoi_id, geom, crop_class, confidence, area_ha, year)` y `features_parcels (id, parcel_id, year, ...)` como dos tablas distintas. **La tabla `parcels` no existe aún en migraciones** (sólo `chat_sessions` y `aois` en `20260511213942_initial_schema.sql`).

**Trade-off**:
- **Opción A (recomendada)**: US-015 crea ambas tablas — `parcels` mínima (id, session_id, aoi_id nullable, geom POLYGON 4326, crop_class TEXT nullable, area_ha REAL, year SMALLINT) + `features_parcels` con FK. Permite poblar `features_parcels` sin esperar a una US específica de "extracción de parcelas".
- **Opción B**: US-015 crea sólo `features_parcels` con `parcel_id BIGINT` sin FK (constraint deferido). Más simple pero acumula deuda — viola §"Critical Rules" del `db/CLAUDE.md` ("Foreign keys con `ON DELETE` explícitos").

**Decisión**: **Opción A**. La US sube una migración previa `create_parcels.sql` y otra `create_features_parcels.sql` con FK. Ambas reversibles. La US no popula `parcels` (eso queda para una US futura de segmentación / GSAA ingest), pero el schema queda íntegro y la siguiente US sólo insertará registros.

### 2.2 Flujo de datos en runtime de la función

```
xr.DataArray (parcel_timeseries)
   coords: time (datetime64), band (NDVI, NDWI, EVI, ...)
   dims:   (time, band)         shape (~30 timesteps, 17 bands)
   attrs:  parcel_id, year
        ▼
1. convert xr → Polars LazyFrame (long format)
   pl.from_arrow(da.to_dataframe().reset_index().to_arrow()).lazy()
        ▼
2. _aggregate_stats: group_by(["parcel_id", "year"]).agg([...9 × 17 columns])
        ▼
3. _fft_harmonics (NDVI, NDWI, EVI): por parcel/year extrae np.array(values),
   aplica np.fft.rfft, devuelve amp/phase de [0..3] → 24 columns
        ▼
4. _detect_phenology (NDVI sólo): SOG/peak/senesc/AUC → 5 columns
        ▼
5. _phenology_slopes (NDVI): pre/post-peak slope + maturity_duration → 3 cols
        ▼
6. join intermediate frames horizontalmente sobre (parcel_id, year)
        ▼
7. .collect(streaming=True) → pl.DataFrame final (~187 columns)
```

### 2.3 Justificación del stack

- **Polars LazyFrame con `streaming=True`**: el corpus de parcelas Italia (~30k parcelas × 5 años × 30 timesteps × 17 índices) excede memoria de laptop dev (estimado ~5 GB en memoria denso, ~800 MB streaming). Cumple regla skill `agrosat-ml-features` §"Polars LazyFrame para datasets grandes (>5 GB)".
- **xarray como entrada**: ya consumido por upstream (`ml/ingest/gee_sampler.py:560 fetch_s2_ndvi_rgb_for_parcel` devuelve xarray). Mantiene homogeneidad de interface.
- **numpy FFT**: scipy.fft introduce dependencia adicional sin ventaja (rfft es idéntico). `np.fft.rfft` ya está disponible vía numpy core.
- **JSONB en `features_parcels`** para `ndvi_stats` y `phenology`: permite consultas ad-hoc en Postgres (`WHERE ndvi_stats->'mean' > 0.5`) sin crear 153 columnas escalares. Columnas escalares se mantienen solo para fenología (consultadas con frecuencia por dashboards).
- **Sin Redis cache en US-015**: la skill recomienda cache para índices espectrales (US-014), pero la agregación temporal es one-shot por (parcel_id, year). Recálculo es barato comparado con I/O del Zarr. Cache se difiere a US-016 si el bulk extractor lo necesita.

## 4. Interfaces públicas (signatures)

### 4.1 `ml/features/temporal_features.py`

```python
import xarray as xr
import polars as pl

def extract_temporal_features(
    parcel_timeseries: xr.DataArray,
    *,
    indices: tuple[str, ...] = (
        "NDVI", "NDWI", "EVI", "NDMI", "NBR", "MSAVI2", "NDRE",
        "MCARI", "CCCI", "GCVI", "PSRI", "NDCI", "FAPAR", "LAI",
        "RENDVI", "SAVI", "TSAVI",
    ),
    fft_indices: tuple[str, ...] = ("NDVI", "NDWI", "EVI"),
    n_fft_harmonics: int = 3,
    sog_threshold: float = 0.3,
    maturity_pct: float = 0.8,
) -> pl.DataFrame:
    """Extrae features temporales por (parcel_id, year) desde una serie xarray.

    Args:
        parcel_timeseries: DataArray con dims (time, band) y attrs
            {"parcel_id": int, "year": int}. Coord `time` debe ser
            datetime64; coord `band` debe contener los nombres de los
            índices listados en `indices`.
        indices: índices espectrales a agregar estadísticamente.
        fft_indices: subset de `indices` al que se aplica descomposición FFT.
        n_fft_harmonics: número de componentes FFT (default 3 → 4 amps y 4
            fases incluyendo componente DC).
        sog_threshold: umbral NDVI para start of greenness (default 0.3).
        maturity_pct: fracción del peak NDVI para definir período de madurez
            (default 0.8 → días con NDVI ≥ 0.8 × peak).

    Returns:
        pl.DataFrame con una fila por (parcel_id, year) y ~187 columnas:
        - 153 stats: {idx}_{stat} para idx ∈ indices, stat ∈
          {mean, std, min, max, p05, p25, p50, p75, p95}
        - 24 FFT: {idx}_fft_amp_{0..3}, {idx}_fft_phase_{0..3} para idx ∈
          fft_indices
        - 8 fenología: sog_doy, peak_doy, peak_value, senescence_doy,
          ndvi_auc, ndvi_slope_pre_peak, ndvi_slope_post_peak,
          maturity_duration_days
        - 2 índice: parcel_id, year

    Raises:
        ValueError: si `parcel_timeseries` no tiene `attrs["parcel_id"]` o
            `attrs["year"]`, si `time` no es datetime64, o si `indices`
            contiene un nombre ausente en `coord band`.
    """
```

### 4.3 Schema PostgreSQL (DDL summary)

```sql
-- {ts}_create_parcels.sql
CREATE TABLE IF NOT EXISTS parcels (
    id BIGSERIAL PRIMARY KEY,
    session_id UUID REFERENCES chat_sessions(id) ON DELETE CASCADE,
    aoi_id BIGINT REFERENCES aois(id) ON DELETE SET NULL,
    geom GEOMETRY(POLYGON, 4326) NOT NULL,
    crop_class TEXT,
    confidence REAL,
    area_ha REAL,
    year SMALLINT NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX parcels_geom_idx ON parcels USING GIST (geom);
CREATE INDEX parcels_session_id_idx ON parcels(session_id);
CREATE INDEX parcels_year_idx ON parcels(year);

-- {ts}_create_features_parcels.sql
CREATE TABLE IF NOT EXISTS features_parcels (
    id BIGSERIAL PRIMARY KEY,
    parcel_id BIGINT NOT NULL REFERENCES parcels(id) ON DELETE CASCADE,
    year SMALLINT NOT NULL,
    alphaearth_embedding VECTOR(64),
    ndvi_stats JSONB NOT NULL DEFAULT '{}'::jsonb,
    phenology JSONB NOT NULL DEFAULT '{}'::jsonb,
    sog_doy SMALLINT,
    peak_doy SMALLINT,
    peak_value REAL,
    senescence_doy SMALLINT,
    ndvi_auc REAL,
    ndvi_slope_pre_peak REAL,
    ndvi_slope_post_peak REAL,
    maturity_duration_days SMALLINT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    CONSTRAINT features_parcels_parcel_year_uniq UNIQUE (parcel_id, year)
);
CREATE INDEX features_parcels_parcel_id_idx ON features_parcels(parcel_id);
CREATE INDEX features_parcels_year_idx ON features_parcels(year);
```

## 9. Riesgos y mitigaciones

| # | Riesgo | Probabilidad | Impacto | Mitigación |
|---|--------|--------------|---------|------------|
| R1 | `parcels` no existe en migrations y otros downstream (US-016, US-018) asumen su existencia | Alta | Alto | US-015 crea `parcels` mínima en su propia migración (Opción A §2.1). Documentado en handoff. Coordinar con Arthur antes de merge para que un US futuro de segmentación amplíe `parcels` sin romper schema |
| R2 | xarray↔Polars conversión costosa en memoria para 30k parcelas × 30 timesteps × 17 bandas | Media | Medio | `streaming=True` en `collect`. Si excede memoria, batch por ROI (procesar Pianura Padana, Toscana, Puglia secuencialmente y concat al final con `pl.concat`) |
| R3 | Fenología no detectable si NDVI ruidoso (caso real Sentinel-2 con nubes residuales) | Media | Medio | Doc en docstring: caller debe pre-filtrar timesteps con QA mask (US-010 EDA Sentinel-2 documentó 6,3 % timesteps inválidos). Función retorna NaN/None graceful, no rompe. Suavizado Savitzky-Golay diferido a US-018 si lo pide rúbrica |
| R4 | `np.fft.rfft` requiere serie temporal regularmente espaciada; Sentinel-2 tiene revisita irregular (~5 días pero con huecos) | Alta | Medio | Pre-interpolación lineal a grid diario antes de FFT — documentar en docstring y test `test_fft_irregular_resamples_first`. Alternativa más robusta: Lomb-Scargle (`scipy.signal.lombscargle`), diferida a backlog si Isaac lo justifica con métrica |
| R5 | Tests integración requieren Postgres+PostGIS+pgvector; CI puede no tenerlos | Media | Bajo | Skip clean con `pytest.importorskip("testcontainers")` o `pytest.skip` si DSN no disponible. Docker image custom para CI (ya planeado en infra). Mientras tanto, `make db-up` local + ejecutar suite manualmente |
| R6 | Coordinación con US-014 (mismo sprint S4): cambios en interfaz xarray de `compute_index` rompen entrada de US-015 | Media | Alto | Contrato explícito documentado: `compute_index → xr.DataArray con dim band ∈ {nombres índices}`. Reunión sync 5 min entre Isaac (US-014) y Isaac (US-015 — mismo owner) al inicio del sprint. Test `test_us014_us015_pipeline_integration` que componga ambas funciones sobre fixture demo |
| R7 | UNIQUE `(parcel_id, year)` vs filas múltiples por temporada (ej. cultivos doble-ciclo) | Baja | Alto | Documentado: la US asume un ciclo vegetativo anual por parcela. Cultivos doble-ciclo (raros en Italia, comunes en Egipto/India) se modelarán en una US futura con `season SMALLINT` añadida vía migración rollforward. No bloquea US-015 |

Cruzar con §13 del v6 si aplica — no hay menciones específicas a US-015 en §13 riesgos.
