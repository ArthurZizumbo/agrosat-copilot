## 0. Resumen ejecutivo

US-013 cierra el Avance 1 con dos entregables consolidados:

1. **Dashboard Streamlit `app/eda_dashboard.py`** con 6 tabs que reflejan las conclusiones de US-010 (Sentinel-2 univariado), US-011 (AlphaEarth + cross-region), US-012 (bivariado + temporal), un mapa folium espacial agregado, y una pestaña final de conclusiones CRISP-ML(Q). Ejecutable con `make eda-dashboard` (atajo a `streamlit run app/eda_dashboard.py`).
2. **Reporte PDF `paper/avance1_eda_report.pdf`** vía `weasyprint` (HTML → PDF, mejor calidad que `reportlab` para texto + figuras embebidas) con las **conclusiones mapeadas a CRISP-ML(Q) Data Understanding**: data sources, quality assessment, integrability, decisions feeding Data Preparation/Feature Engineering.

**Decisión pragmática (12-may-2026)**: Streamlit es **stateless de archivos** — las figuras y tablas que muestra son las **mismas PNG/CSV exportadas por los 3 notebooks** (`paper/figures/us-010/`, `us-011/`, `us-012/`). El dashboard no re-computa nada: lee del filesystem. Esto desacopla US-013 del estado de ejecución de los notebooks: una vez que cualquier notebook exporta sus figuras, su tab del dashboard se llena automáticamente.

**Posicionamiento en el calendario**: US-013 se trabaja **en paralelo** con US-010/011/012. El esqueleto Streamlit (6 tabs con placeholders) se puede escribir el primer hora del sprint. El contenido (figuras+tablas+conclusiones) se llena a medida que cada notebook cierra. Esto reduce el cuello de botella secuencial que el plan v6 implícitamente asumía.

**No bloqueante para el A1 PDF**: si el dashboard Streamlit tiene bugs el día de entrega, el **PDF tiene prioridad absoluta** (es el artefacto evaluado, no la app). El dashboard es valor agregado para el sponsor (Dr. Camacho) y para defensa oral.

### 2.3 Por qué Streamlit y no FastAPI + Nuxt

Es tentador integrarlo al frontend Nuxt 4 existente. Lo evito porque:

1. **El frontend Nuxt es el cliente del agente conversacional (US-018+)**, su scope es chat + mapa + AOI drawer + LLM switcher. Mezclar un dashboard EDA contamina su responsibility.
2. **Streamlit es 100 líneas vs ~500 líneas** de Vue componentes + composables + Pinia stores que harían lo mismo. Para un dashboard interno (sponsor + equipo, no usuarios finales), el costo de desarrollo en Vue es excesivo.
3. **Streamlit no es producción**: es una herramienta de exploración. Después del Avance 1, el dashboard puede archivarse o convertirse en notebook si nadie lo usa. La inversión es barata.
4. El plan v6 lo pide literal en la rúbrica (línea 1060: "Dashboard Streamlit").

### 2.4 Por qué weasyprint y no reportlab

Tres razones técnicas:

1. **weasyprint usa HTML+CSS como input**: el mismo template Jinja2 puede generar HTML browsable y PDF impreso. Reutilizable después para reportes posteriores (Avance 5, Avance 7).
2. **reportlab requiere descripción imperativa del layout** (canvas, drawing primitives). Para texto + figuras + tablas, weasyprint es 5x más conciso.
3. **weasyprint maneja paginación automática** + table of contents + numeración. reportlab requiere implementar todo manualmente.

Trade-off: weasyprint pesa más como dependencia (~200 MB con cairo + pango) y tiene rough edges en Windows. Mitigación: lo agregamos al grupo `paper` opcional, y CI corre en Ubuntu (no Windows) para el render PDF.

## 10. Decisiones tomadas en este plan

| Decisión | Razón |
|----------|-------|
| **Streamlit en lugar de Nuxt** | El frontend Nuxt es para el agente conversacional, no para EDA interno · Streamlit es 5x menos código para el mismo resultado |
| **weasyprint en lugar de reportlab** | HTML+CSS reutilizable · paginación automática · maneja tablas+figuras+texto idiomáticamente |
| **Dashboard file-driven (no re-computa)** | Desacopla US-013 del estado de US-010/011/012 · permite trabajar en paralelo · cada tab se llena cuando su notebook cierra |
| **Puerto 8501** (default Streamlit) | Consistente con patrón de puertos custom 8010/3010/etc; no choca con FastAPI 8010 ni Nuxt 3010 |
| **AC-10 (re-run desde dashboard) opcional** | Useful demo trick pero no rúbrica; si bloquea, lo dejamos como TODO |
