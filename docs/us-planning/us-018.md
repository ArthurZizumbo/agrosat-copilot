### 2.1 Decisiones tecnicas clave

| # | Decision | Justificacion | Trade-off rechazado |
|---|----------|---------------|---------------------|
| D1 | **Split espacial = folds oficiales PASTIS-R 1-5** (no GroupKFold custom, no SpatialKFold geografico nuevo) | Folds 1-5 ya son disjuntos espacialmente por construccion (Sainte-Fare-Garnot 2021), ya cargados desde `ml/ingest/pastis_loader.py`, **son la referencia academica del dataset**. Comparable con la literatura | GroupKFold por tile geografico custom: introduce divergencia con literatura, riesgo de leakage si la geometria del grouping no se valida |
| D2 | **scikit-learn ^1.8.0** para todos los filtros/extractores salvo UMAP (umap-learn ^0.5.7) y XGBoost (xgboost ^3.2.0) | Ya estan en `pyproject.toml` (lineas 81-104). Cero deps nuevas | scipy.stats directo para chi2/ANOVA: re-implementar sin valor anadido |
| D3 | **PCA con `target_variance=0.95`** parametrico, no `n_components` fijo | Cumple literal CA "objetivo 95%". Numero de componentes depende del nivel de redundancia post-filtrado | `n_components=64` arbitrario: rompe la trazabilidad a varianza |
| D4 | **UMAP solo 2D, no 3D, no para training** | CA literal "UMAP 2D para visualizacion". Es prepro/EDA, no feature engineering productivo (UMAP es no-determinista entre versiones y costoso para inference) | UMAP nD para baseline: NO recomendado por la propia documentacion umap-learn para downstream ML |
| D5 | **Polars in / Polars out** en todas las funciones publicas + adaptadores numpy internos | Cumple regla `ml/CLAUDE.md` "NEVER usar pandas cuando Polars puede hacerlo". sklearn requiere numpy, asi que conversion explicita en helper privado `_to_numpy(df, exclude=("parcel_id", "year"))` | Pandas como tipo de retorno: viola CLAUDE.md `NEVER pandas` |
| D6 | **chi2 con fallback sintetico documentado** si no hay categoricas reales | CA literal "chi-cuadrado para categoricas". Las 187 features upstream son todas numericas. Cumplir el CA discretizando `peak_doy` en 4 cuartiles como prueba metodologica es legitimo y se documenta explicitamente en notebook §3.3 | Saltar el CA: incumple rubrica Avance 2 |
| D7 | **Feature importance con RF + XGB baseline NO production** (params ligeros) | Es complemento metodologico del CA, no el baseline de US-018 v6 (RF/XGB sobre AlphaEarth). Hyperparams documentados como "exploratorios" | Re-usar el baseline production: requiere primero cerrar US-018 v6 → bloqueo circular |
| D8 | **Reportes en `reports/feature_selection/`** + solo 2 PNG commiteados | Permite reproducir tablas/figuras desde el notebook + commit ligero de los artefactos visuales mas representativos para revision sin ejecutar el notebook | Commitear todos los CSV: bloat del repo (se regeneran en CI con papermill) |
| D9 | **Normalizacion: `ColumnTransformer` por familia de modelo, no scaler unico global** | Modelos lineales y NN tienen necesidades distintas. CA literal: "StandardScaler para lineales, MinMax para NN, Yeo-Johnson para sesgadas, log para LAI/biomasa" | Un scaler global: rompe la justificacion por familia |
| D10 | **Yeo-Johnson sobre NDVI (no Box-Cox)** | NDVI puede ser negativo (agua, sombras); Box-Cox solo acepta positivos | Box-Cox forzando shift: introduce parametro arbitrario sin base academica |

### 2.2 Flujo de datos en runtime

```
1. Carga X, y, folds desde PASTIS-R subset (helper `_load_pastis_features_subset`)
   - Usa pastis_loader + extract_temporal_features (US-015) + spectral_indices (US-014)
   - Cachea en `data/test_fixtures/feature_selection_subset.parquet` (~500 muestras × 187 cols)

2. EDA inicial: dist plots + skew + missing report
   (re-usa helpers de ml/analysis/distributions.py si aplica)

3. FILTROS:
   X1 = apply_variance_threshold(X, 0.01)       → drop ~3-10 cols
   X2 = drop_correlated_features(X1, 0.95)      → drop ~10-20 cols (cluster NDVI/NDRE/NDWI/SAVI)
   chi2_select(X2_binned, y, k=20)              → reporta top-20 (notebook §3.3)
   anova_f_select(X2, y, k=20)                  → reporta top-20 (CSV)

4. EXTRACTORES (sobre X2 escalado con StandardScaler):
   X_scaled = StandardScaler().fit_transform(X2)
   pca = fit_pca(X_scaled, target_variance=0.95) → n_components
   fa  = fit_factor_analysis(X_scaled, n_factors=5)
   emb = fit_umap_2d(X_scaled, y, random_state=42)

5. COMPLEMENTO:
   compute_feature_importance(X, y, model="rf")   → top-20
   compute_feature_importance(X, y, model="xgb")  → top-20
   overlap_top10 = set(rf.top10) & set(xgb.top10)

6. COMPARATIVA antes/despues (con PASTIS folds 1-5):
   compare_before_after(X_raw=X, X_filtered=X2, X_pca=pca_transform(X_scaled), y, folds)
   → 4 estrategias × (F1-macro mean±std, mIoU mean±std)
   → before_after.csv + before_after.md

7. NORMALIZACION:
   decisions = [select_normalizer(name, stats) for name, stats in per_feature_stats.items()]
   preprocessor_linear = make_preprocessor(strategy="linear", df=X2)
   preprocessor_nn     = make_preprocessor(strategy="nn", df=X2)
   → normalization_decisions.csv

8. CONCLUSIONES CRISP-ML(Q) (markdown cell)
```

### 2.3 Justificacion del stack

- **scikit-learn 1.8**: ya esta en `pyproject.toml`. `VarianceThreshold`, `chi2`, `f_classif`, `PCA`, `FactorAnalysis`, `RandomForestClassifier`, `StandardScaler`, `MinMaxScaler`, `PowerTransformer`, `ColumnTransformer` todos disponibles.
- **xgboost 3.2**: ya esta en `pyproject.toml`. `XGBClassifier(tree_method="hist")` rapido en CPU.
- **umap-learn 0.5.7**: ya esta en `pyproject.toml`. `UMAP(n_components=2, random_state=42)`.
- **Polars 1.x**: regla `ml/CLAUDE.md`. Conversion explicita a numpy en `_to_numpy` privado.
- **matplotlib + seaborn**: para PNG (heatmap correlacion, scree plot, UMAP scatter). Ya disponibles via spyndex deps transitive.
- **scipy.stats.skew**: para deteccion de sesgo en regla `select_normalizer`.

import polars as pl
import numpy as np
from sklearn.compose import ColumnTransformer
from typing import Literal

# === FILTERS ===

def apply_variance_threshold(
    df: pl.DataFrame,
    threshold: float = 0.01,
    exclude_cols: tuple[str, ...] = ("parcel_id", "year"),
) -> tuple[pl.DataFrame, dict[str, list[str] | dict[str, float]]]:
    """Elimina features con varianza por debajo del umbral.

    Returns:
        (df_filtered, report) donde report tiene
        {"kept": [...], "removed": [...], "variances": {col: var}}.
    """

def drop_correlated_features(
    df: pl.DataFrame,
    threshold: float = 0.95,
    method: Literal["pearson", "spearman"] = "pearson",
    exclude_cols: tuple[str, ...] = ("parcel_id", "year"),
) -> tuple[pl.DataFrame, dict[str, list[str] | np.ndarray]]:
    """Remueve un feature de cada par con |r| > threshold (deterministic order)."""

def chi2_select(
    X: pl.DataFrame,
    y: pl.Series,
    k_best: int = 20,
    *,
    binning_strategy: Literal["quartiles", "deciles"] | None = "quartiles",
) -> tuple[pl.DataFrame, dict[str, float]]:
    """Chi-cuadrado para categoricas. Si X es numerico continuo, hace binning
    documentado segun `binning_strategy` (default quartiles).

    Returns:
        (top_k_features_df, {"feature": chi2_stat, ...}).
    """

def anova_f_select(
    X: pl.DataFrame,
    y: pl.Series,
    k_best: int = 20,
    exclude_cols: tuple[str, ...] = ("parcel_id", "year"),
) -> tuple[pl.DataFrame, dict[str, float]]:
    """ANOVA F-score para numericas."""

# === EXTRACTORS ===

def fit_pca(
    X_scaled: np.ndarray,
    target_variance: float = 0.95,
    *,
    random_state: int = 42,
) -> dict[str, np.ndarray | int | float]:
    """PCA hasta acumular `target_variance` de varianza explicada.

    Returns:
        {"n_components": int, "components": ndarray, "explained_variance_ratio":
        ndarray, "cumulative_variance": ndarray, "transformer": PCA}.
    """

def fit_factor_analysis(
    X_scaled: np.ndarray,
    n_factors: int = 5,
    *,
    random_state: int = 42,
) -> dict[str, np.ndarray]:
    """Factor Analysis para descubrir factores latentes (firmas espectrales)."""

def fit_umap_2d(
    X_scaled: np.ndarray,
    y: np.ndarray | None = None,
    *,
    n_neighbors: int = 15,
    min_dist: float = 0.1,
    random_state: int = 42,
) -> np.ndarray:
    """UMAP 2D para visualizacion. Devuelve embedding (n_samples, 2)."""

# === COMPLEMENT ===

def compute_feature_importance(
    X: pl.DataFrame,
    y: pl.Series,
    *,
    model: Literal["rf", "xgb"] = "rf",
    n_estimators: int = 200,
    random_state: int = 42,
    n_jobs: int = -1,
    exclude_cols: tuple[str, ...] = ("parcel_id", "year"),
) -> pl.DataFrame:
    """Importancia de features con RF (Gini) o XGB (Gain).

    Returns:
        pl.DataFrame con columnas (feature, importance, rank) ordenado desc.
    """

# === COMPARATIVE ===

def compare_before_after(
    X_raw: pl.DataFrame,
    X_selected: pl.DataFrame,
    y: pl.Series,
    folds: np.ndarray,
    *,
    extra_strategies: dict[str, pl.DataFrame] | None = None,
    random_state: int = 42,
) -> pl.DataFrame:
    """CV con folds PASTIS 1-5 sobre 4 estrategias. Devuelve tabla
    (strategy, n_features, f1_macro_mean, f1_macro_std, miou_mean, miou_std).
    """

# === NORMALIZATION ===

def select_normalizer(
    feature_name: str,
    distribution_stats: dict[str, float],
    *,
    strategy: Literal["linear", "nn"] = "linear",
) -> tuple[str, str]:
    """Decide scaler segun nombre/distribucion.

    Returns:
        (scaler_name, justification_short).

    Reglas:
        - feature in {"LAI", "biomass_*"}        → log1p (strictly positive)
        - skew > 1.0 (e.g. NDVI sesgado)         → yeo-johnson (handles negatives)
        - strategy == "nn"                       → minmax
        - strategy == "linear"  (default)        → standard
    """

def make_preprocessor(
    df: pl.DataFrame,
    *,
    strategy: Literal["linear", "nn"] = "linear",
    exclude_cols: tuple[str, ...] = ("parcel_id", "year"),
) -> ColumnTransformer:
    """ColumnTransformer ruteado segun select_normalizer por feature."""
