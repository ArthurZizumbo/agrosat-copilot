# Manual Test — US-016 · Fusión multisensor a nivel parcela

**Epic**: E3 · **Avance**: A2 · **Sprint**: S4 · **Estado**: coding (paralelo `ml-engineer` + `mlops-engineer`)

> US-016 es 100 % ML + DB-lectura + Dagster (sin UI, sin chat, sin LLM). La
> mayoría de los criterios se validan automáticamente vía `pytest`. Esta guía
> cubre los flujos que requieren juicio humano: ejecutar el script CLI
> end-to-end, materializar los assets Dagster, validar contra GEE real,
> versionar con DVC y aplicar el scaler a un frame de inferencia downstream.

- Repo clonado y `poetry install` ejecutado con grupos `dev,test,ml,geo,dagster`.
- Python 3.12.x + Poetry ≥ 1.7.
- `.env.local` con `GEE_PROJECT`, `GEE_SERVICE_ACCOUNT_JSON_PATH`,
  `DATABASE_URL`, `DVC_GCS_CREDENTIALS_PATH` para los flujos 3 y 4.

  (~3 KB, 9 parcelas: 3 por región — Pianura Padana, Toscana, Puglia).

  y DVC 3.48+ instalado (`poetry run dvc --version`).

## Comandos exactos para versionar al cierre (Isaac)

Una vez los 3 outputs estén materializados y validados, ejecutar como parte
de la fase de cierre US-016:

```bash
# 1. Versionar con DVC
dvc add data/features/features_fused_v1.parquet
dvc add data/splits/spatial_kfold_v1/
dvc add artifacts/scaler_v1.pkl

# 2. Commit de los .dvc files
git add data/features/features_fused_v1.parquet.dvc \
        data/splits/spatial_kfold_v1.dvc \
        artifacts/scaler_v1.pkl.dvc \
        .gitignore
git commit -m "data(E3): track US-016 multisensor outputs (features_fused, kfold, scaler)"

# 3. Push al remoto GCS
make dvc-push

# 4. Tags semánticos
git tag fused-features-italy-v1 -m "US-016 fused features v1 (189 cols sin FarSLIP)"
git tag spatial-kfold-italy-v1 -m "US-016 spatial K-fold v1 (5 folds, h3 res 5, buffer 1km)"
git tag scaler-v1 -m "US-016 StandardScaler v1 (fold_0 train)"
git push origin --tags
```

> No se ejecuta `dvc add` en este punto del coding porque los parquet reales
> requieren GEE para los bloques S1/SRTM/ERA5/AlphaEarth. El asset Dagster
> y el script CLI los generan en flujos 1, 2, 3.
