## Test 3 — Wrapper Earth Engine (requiere credenciales GEE)

**Objetivo**: confirmar que `compute_index_ee` funciona contra una `ee.Image`
real, no solo contra el mock del suite automático.

**Pasos**:

1. Autenticación (una de las dos vías):
   - `earthengine authenticate` (cuenta personal o de servicio), o
   - `gcloud auth application-default login` + `gcloud config set project <proyecto-gcp>`.
2. `poetry run python`:

   ```python
   import ee
   ee.Initialize(project="<tu-proyecto-gcp>")
   from ml.features import compute_index_ee
   img = (ee.ImageCollection("COPERNICUS/S2_SR_HARMONIZED")
            .filterDate("2024-06-01", "2024-06-15")
            .filterBounds(ee.Geometry.Point([12.49, 41.89]))
            .first())
   ndvi_img = compute_index_ee(img, "NDVI")
   print(ndvi_img.bandNames().getInfo())
   ```

**Resultado esperado**:

- Lista de bandas devuelta por `getInfo()` incluye `"NDVI"`.
- Sin `ImportError` (eemont instalado) ni `EEException` (credenciales OK).

(`poetry install --with ml`) y reautenticar GEE.

## Test 5 — Cache Redis end-to-end con Redis real (opcional)

**Objetivo**: el suite usa `fakeredis`; este test usa Redis real para confirmar
que la serialización pickle no tiene incompatibilidades de versión.

**Pasos**:

1. Levantar Redis local: `docker run --rm -p 6379:6379 redis:7-alpine` (otra terminal).
2. `poetry run python`:

   ```python
   import redis
   from ml.features import compute_index_cached
   from tests.ml.features.fixtures.synthetic_pixels import make_forest_pixel

   r = redis.Redis(host="localhost", port=6379, decode_responses=False)
   da = make_forest_pixel()
   miss = compute_index_cached(da, "NDVI", scene_id="manual_test", redis_client=r)
   hit  = compute_index_cached(da, "NDVI", scene_id="manual_test", redis_client=r)
   assert float(miss.mean()) == float(hit.mean())
   print("Hit/miss OK:", float(hit.mean()))
   print("Cache key TTL:", r.ttl("manual_test:NDVI"))
   ```

**Resultado esperado**:

- Sin `AssertionError`.
- TTL impreso ≈ `86400` (24 h por defecto).

**Resultado real 2026-05-16** — ✅ **PASS**

- Contenedor temporal: `docker run --rm -d --name us014-redis-test -p 6379:6379 redis:7-alpine`.
- Salida: `miss_mean= 0.7999999523162842 hit_mean= 0.7999999523162842 ttl= 86400 exists= 1`.
- Miss y hit devuelven el mismo valor (NDVI=0.80 sobre fixture forest), TTL=86400s (24h),
  la clave `manual_test:NDVI` persiste en Redis.
- Contenedor detenido al terminar (`docker stop us014-redis-test`).
