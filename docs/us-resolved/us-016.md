## Resumen ejecutivo

US-016 entrega el **vector tabular fusionado por parcela** que alimenta los baselines RF/XGBoost (US-019/020/021) y los heads tabulares de las arquitecturas de segmentacion (EPIC 5). Dado un `GeoDataFrame` con `parcel_id`, `year`, `geometry` (POLYGON EPSG:4326), [`build_fused_features`](../../ml/features/fusion.py) produce un `pl.DataFrame` Polars 1.x con **191 columnas** por `(parcel_id, year)` — 189 features + 2 indice:

- **64 cols AlphaEarth** (`ae_00..ae_63`): media del embedding de 64 dims sobre la parcela.
- **85 cols indices x stats** (`{idx}_{stat}`): 17 indices canonicos x 5 stats temporales (`mean, std, p25, p50, p95`).
- **10 cols Sentinel-1** (`s1_vv_*`, `s1_vh_*`): backscatter VV+VH x 5 stats con preset GRD IW asc+desc despeckle Lee 7x7 sigma0 dB.
- **3 cols SRTM** (`srtm_elev_mean`, `srtm_slope_mean`, `srtm_aspect_dominant`): elevacion + slope + aspect dominante string cardinal (8 cuadrantes).
- **24 cols ERA5 mensual** (`era5_tmean_m01..m12`, `era5_prec_m01..m12`): temperatura media + precipitacion acumulada por mes, oversampling `scale=1000` para soportar parcelas sub-pixel (fix Bug-6).
- **3 cols geometria** (`geom_area_ha`, `geom_perimeter_m`, `geom_elongation`): Polsby-Popper inverso, reproyectado a EPSG:3857 para unidades metricas.

**Bloque FarSLIP opcional** (`farslip_000..farslip_511`, +512 dims = 701 features) via LEFT JOIN cuando `data/farslip/embeddings_italy.parquet` exista (US-016b paralela). Si no existe, log estructurado `farslip_block_skipped` y se omite el bloque sin fallar.

La fusion vive en **Polars `LazyFrame` con `collect(engine="streaming")`** (firma Polars 1.x, sustituye `streaming=True` deprecado del plan). El orden final garantizado por `.sort("parcel_id")` da determinismo byte-exacto entre re-ejecuciones (MD5 invariante demostrado en 3 corridas del manual-test).

[`build_spatial_kfold`](../../ml/features/spatial_split.py) genera **K=5 folds espaciales** con tessellation H3 res 5 (~252 km²/celda) + KMeans sobre centroides + buffer de exclusion de 1 km. Garantiza zero leakage cross-fold y class balance < 3x (validado con grid sintetica 100 parcelas).

[`fit_scaler_on_train`](../../ml/features/scaler.py) ajusta un `StandardScaler` joblib **solo sobre el split train del Fold-0** con validacion explicita de no-leakage (raise si `train_ids` interseca `val_ids` o `test_ids`). Filtra silenciosamente columnas categoricas (`srtm_aspect_dominant`) y all-NaN (modo demo sin GEE) con log estructurado.

La orquestacion dual (CLI Typer + 3 assets Dagster) comparte un unico entry point `build_fused_features` para garantizar DRY: los outputs del script y de los assets son **byte-identicos** (MD5 `4c58c36f7d01c8da998f10292806066d` verificado en QA Fase 4.2).

Durante el ciclo de QA se descubrieron y resolvieron **6 bugs** in-branch (3 detectados en QA Fase 4 sobre Makefile/CRS/scaler/doc, 1 al validar el primer fix, 1 sub-pixel ERA5 detectado solo con GEE real en QA Fase 4.2). La US queda con **140 tests pass + 3 skipped** (`requires_gee` por dev sin creds EE), `cd ml && ruff check .` limpio, y AC-1..AC-13 + AC-15 verdes. AC-14 (DVC push real) deferido formalmente a sub-US **US-016.1** ya en el backlog.

### Multi-tenancy y `session_id`

US-016 es **100% ML batch + DB lectura + Dagster**; **no expone endpoints HTTP** ni codigo de runtime que consulte `parcels`/`features_parcels`. La proteccion multi-tenant se hereda del schema US-015:

- `parcels.session_id UUID REFERENCES chat_sessions(id) ON DELETE CASCADE` (clave de tenancy).
- `features_parcels.parcel_id BIGINT REFERENCES parcels(id) ON DELETE CASCADE` (heredada via JOIN).

**Pendiente para US futuras** (NO bloqueante para US-016): cuando US-018+ expongan los features fusionados en endpoints `/parcels/*` o `/features/*`, deben filtrar por `session_id` via subquery a `parcels` (mismo patron que US-015) y cerrar **W-5** (RLS heredado del initial schema). Documentado en handoff US-015 §Multi-tenancy.
